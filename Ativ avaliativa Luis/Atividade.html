Gerenciamento de Configuração de Software

Controle de Versão

O controle de versão é uma peça fundamental para o desenvolvimento de software, tem como objetivo a ajudar a rastrear as mudanças feitas no código por uma equipe. Essa funcionalidade somente é possível com o uso de ferramentas como GIT, SVN e Mercurial onde você pode ver as alterações, por quem foi feita e quando.

Controle de Mudanças

Para ajudar a manter a integridade e qualidade do software utiliza-se o controle de mudancas que permite a gestão das alterações no projeto, assim documentando as modificações, revisando e aprovando-as antes da implementação.

Integração Contínua

Integração contínua, ou ci, é uma técnica DevOps para automatizar a integração de alterações de código feitas por vários desenvolvedores em um único repositório central.há ferramentas como Jenkins, travis CI, github, que automatizam a construção e o teste, garantindo que o código esteja funcionando bem a maior parte do tempo.
